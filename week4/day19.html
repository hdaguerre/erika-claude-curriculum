The Skill Creator — Building a Skill from Scratch

Estimated time: 3 hours
ConceptsConceptos

1. The skill-creator in practice1. El skill-creator en la práctica

The skill-creator is a skill that helps you build other skills. It's a meta-skill — a skill whose job is to produce more skills. Every project has it as standard infrastructure.

You run it by typing /skill-creator in a Claude Code session inside the erika project. It walks you through a structured process: capturing what the skill should do, writing the SKILL.md, creating test cases (evals.json), and validating the result. By the end, you have a working skill file you didn't have to write by hand.

El skill-creator es una habilidad que ayuda a construir otras habilidades. Es una meta-habilidad — una habilidad cuyo trabajo es producir más habilidades. Todos los proyectos la tienen como infraestructura estándar.

La ejecutás escribiendo /skill-creator en una sesión de Claude Code dentro del proyecto erika. Te guía a través de un proceso estructurado: capturar qué debe hacer la habilidad, escribir el SKILL.md, crear casos de prueba (evals.json) y validar el resultado. Al final, tenés un archivo de habilidad que funciona sin tener que escribirlo a mano.

SQL Connection

The skill-creator is a code generator — like a wizard in SQL Developer that scaffolds a stored procedure template based on your inputs. You tell it the procedure name, the parameters, and the logic you want. It generates the structure. You still review and edit it, but the boilerplate is built for you. Same idea here: describe the skill, and the scaffold appears.

Conexión SQL

El skill-creator es un generador de código — como un asistente en SQL Developer que crea el scaffolding de una plantilla de procedimiento almacenado basándose en tus entradas. Vos igual la editás, pero la estructura ya está construida.

2. Skill anatomy — revisited with context2. Anatomía de una habilidad — revisitada con contexto

Now that you've seen workflows and tools, the skill concept clicks differently. A skill is like a mini-workflow for how Claude behaves in a specific context. It's triggered by your request, loaded into Claude's working context, and followed as instructions for that session.

Skills live in skills/[skill-name]/SKILL.md. The file has two parts:

  • YAML front matter — the name and description fields. This is the skill's identity and its trigger.
  • SKILL.md body — the actual instructions Claude follows when the skill is active. Process steps, tone guidance, output format.

Ahora que viste workflows y herramientas, el concepto de habilidad encaja de otra manera. Una habilidad es como un mini-workflow sobre cómo se comporta Claude en un contexto específico. Se activa con tu solicitud, se carga en el contexto de Claude y se sigue como instrucciones para esa sesión.

Las habilidades viven en skills/[nombre-habilidad]/SKILL.md. El archivo tiene dos partes:

  • Encabezado YAML — los campos name y description. Esta es la identidad de la habilidad y su disparador.
  • Cuerpo del SKILL.md — las instrucciones reales que Claude sigue cuando la habilidad está activa. Pasos del proceso, guía de tono, formato de salida.

3. What makes a good skill description3. Qué hace que una descripción de habilidad sea buena

The description field in the YAML header is the most important part of any skill. It's the trigger mechanism — it tells Claude when to load and follow this skill. If it's vague, Claude won't know when to use it. If it's precise, it triggers reliably.

Compare these two:

  • Bad: "Use for writing."
  • Good: "Use when the user wants to write an Instagram caption for food or bakery content in Spanish — includes caption body, hashtag set, and tone guidance for a warm, personal bakery brand voice."

The good version names: the action (write a caption), the content type (food/bakery), the language (Spanish), and the key output elements. Claude can match that to almost any bakery-related request. The bad version could match anything — so it matches nothing reliably.

El campo description en el encabezado YAML es la parte más importante de cualquier habilidad. Es el mecanismo disparador — le dice a Claude cuándo cargar y seguir esta habilidad. Si es vago, Claude no va a saber cuándo usarlo. Si es preciso, se activa de forma confiable.

Compará estos dos:

  • Malo: "Usar para escribir."
  • Bueno: "Usar cuando la usuaria quiere escribir un caption de Instagram para contenido de comida o repostería en español — incluye cuerpo del caption, conjunto de hashtags y guía de tono para una voz de marca de panadería cálida y personal."

La versión buena nombra: la acción (escribir un caption), el tipo de contenido (comida/repostería), el idioma (español) y los elementos clave de salida. Claude puede vincular eso con casi cualquier solicitud relacionada con la panadería. La versión mala puede coincidir con cualquier cosa — así que no coincide con nada de forma confiable.

Key Insight

Write the description as if you're briefing a new assistant who has never met you. They need to know: what kind of request triggers this, what they should do, and what the output should look like. Specific is kind. Vague is useless.

Idea clave

Escribí la descripción como si le estuvieras explicando a un asistente nuevo que nunca te conoció. Necesitan saber: qué tipo de solicitud activa esto, qué deben hacer y cómo debe verse el resultado. Lo específico es amable. Lo vago es inútil.

  1. Open Claude Code in the erika project. In your terminal:
    cd /Users/hdaguerre/Desktop/claude-projects/erika && claude
    Abrí Claude Code en el proyecto erika. En tu terminal:
    cd /Users/hdaguerre/Desktop/claude-projects/erika && claude
  2. Type /skill-creator and press Enter. The skill-creator will start and ask you questions. Answer them thoughtfully — your answers are what make the skill good.
    Escribí /skill-creator y presioná Enter. El skill-creator va a arrancar y hacerte preguntas. Respondelas con cuidado — tus respuestas son las que hacen que la habilidad sea buena.
  3. You're building a skill called bakery-caption. When the skill-creator asks what the skill should do, describe it like this:

    "This skill is for writing Instagram captions for my bakery, Mi Taller. The captions are in Spanish, warm and personal, and highlight handmade quality. The output includes a caption body, 5–8 relevant hashtags in Spanish and English, and an optional emoji suggestion. It should feel like it was written by a person who loves what she makes — not a marketing bot."

    Use your own words, but be that specific.
    Estás construyendo una habilidad llamada bakery-caption. Cuando el skill-creator pregunte qué debe hacer la habilidad, describila así:

    "Esta habilidad es para escribir captions de Instagram para mi panadería, Mi Taller. Los captions están en español, son cálidos y personales, y destacan la calidad artesanal. El resultado incluye el cuerpo del caption, 5–8 hashtags relevantes en español e inglés, y una sugerencia de emoji opcional. Debe sentirse como si lo hubiera escrito una persona que ama lo que hace — no un bot de marketing."

    Usá tus propias palabras, pero sé así de específica.
  4. Let the skill-creator guide you through the rest: the full description, the SKILL.md body, and the test cases. When it proposes something you don't like, say so. This is a conversation, not a form.
    Dejá que el skill-creator te guíe por el resto: la descripción completa, el cuerpo del SKILL.md y los casos de prueba. Cuando proponga algo que no te guste, decilo. Esto es una conversación, no un formulario.
  5. When the skill is created, find it: skills/bakery-caption/SKILL.md
    Open the file. Read the description field in the YAML header. Ask yourself: "If someone described a photo of my alfajores in a new chat session, would this description be specific enough to trigger the skill?" If not, edit it until it is.
    Cuando la habilidad esté creada, encontrala: skills/bakery-caption/SKILL.md
    Abrí el archivo. Leé el campo de descripción en el encabezado YAML. Preguntate: "Si alguien describiera una foto de mis alfajores en una nueva sesión de chat, ¿sería esta descripción lo suficientemente específica para activar la habilidad?" Si no, editala hasta que lo sea.
  6. Quick test: In a new Claude Code message (still in the erika project), describe a baked good and ask for a caption. Did the bakery-caption skill activate? Did the output sound like you? If not, open the SKILL.md and refine the instructions.
    Prueba rápida: En un nuevo mensaje de Claude Code (todavía en el proyecto erika), describí un producto de panadería y pedí un caption. ¿Se activó la habilidad bakery-caption? ¿El resultado sonó como vos? Si no, abrí el SKILL.md y refiná las instrucciones.
