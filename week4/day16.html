Workflows — Writing SOPs Claude Follows

Estimated time: 2.5 hours
ConceptsConceptos

1. What is a workflow?1. ¿Qué es un workflow?

A workflow is a plain-language SOP (standard operating procedure) that Claude reads before executing a task. It lives in workflows/ as a markdown file. It tells Claude: what the goal is, what inputs are required, which tools to use and in what order, what to do when something fails, and what the expected output looks like.

Claude reads the workflow and orchestrates the execution. It doesn't improvise — it follows your instructions, step by step, the way you wrote them.

Un workflow es un SOP (procedimiento operativo estándar) en lenguaje simple que Claude lee antes de ejecutar una tarea. Vive en workflows/ como archivo markdown. Le dice a Claude: cuál es el objetivo, qué entradas se requieren, qué herramientas usar y en qué orden, qué hacer cuando algo falla, y cómo se ve el resultado esperado. Claude lo lee y orquesta la ejecución.

SQL Connection

A workflow is a query plan written in plain language. In Oracle, the query optimizer decides execution order automatically. Here, Claude is the optimizer — it reads your workflow and decides how to execute each step. And just like a stored procedure has an EXCEPTION block? Your workflow's Error Handling section is exactly that.

Conexión SQL

Un workflow = un plan de consulta escrito en lenguaje simple. En Oracle, el optimizador de consultas decide el orden de ejecución. Aquí, Claude es el optimizador — lee el workflow y decide cómo ejecutar cada paso. El manejo de errores en un workflow = el bloque EXCEPTION de Oracle en un procedimiento almacenado.

2. Writing good workflows2. Cómo escribir buenos workflows

A workflow is not code. It's English — or Spanish. It should be specific enough that Claude can follow it step by step, but not so prescriptive that it can't adapt when something unexpected happens.

A well-structured workflow has five sections:

  • Objective — What is this workflow trying to accomplish?
  • Required Inputs — What does Claude need before it can start?
  • Steps — Numbered sequence of what to do, in order
  • Error Handling — What to do when something goes wrong
  • Expected Output — What does success look like? Include an example.

Un workflow NO es código. Es inglés (o español). Debe ser suficientemente específico para que Claude pueda seguirlo paso a paso, pero no tan prescriptivo que no pueda adaptarse cuando algo inesperado ocurre.

Un workflow bien estructurado tiene cinco secciones:

  • Objetivo — ¿Qué intenta lograr este workflow?
  • Entradas Requeridas — ¿Qué necesita Claude antes de empezar?
  • Pasos — Secuencia numerada de qué hacer, en orden
  • Manejo de Errores — Qué hacer cuando algo sale mal
  • Resultado Esperado — ¿Cómo se ve el éxito? Incluí un ejemplo.
Key Insight

The "Required Inputs" section is the most commonly skipped — and the most important. If Claude doesn't know what it needs before it starts, it will either guess or stop to ask. Both are bad. Define your inputs like you're writing a procedure signature.

Idea Clave

La sección "Entradas Requeridas" es la que más se saltea — y la más importante. Si Claude no sabe qué necesita antes de empezar, va a adivinar o va a detenerse a preguntar. Las dos opciones son malas. Definí tus entradas como si escribieras la firma de un procedimiento.

3. Workflow vs. Skill — an important distinction3. Workflow vs. Habilidad — una distinción importante

These two concepts are easy to confuse. Here's the clean version:

  • A Skill tells Claude how to respond to a certain type of request — it's about Claude's behavior, tone, and output format.
  • A Workflow tells Claude how to execute a multi-step process — it's about sequence, tools, and what to do when steps fail.

A skill says "when someone asks for a bakery caption, do it this way." A workflow says "when generating a recipe post, fetch this, check that, run this tool, then produce this output."

Es fácil confundir estos dos conceptos. Acá la versión clara:

  • Una Habilidad le dice a Claude cómo responder a cierto tipo de solicitud — es sobre el comportamiento, el tono y el formato de salida de Claude.
  • Un Workflow le dice a Claude cómo ejecutar un proceso de varios pasos — es sobre la secuencia, las herramientas y qué hacer cuando los pasos fallan.

Una habilidad dice "cuando alguien pide un caption para la panadería, hacelo así." Un workflow dice "cuando generás una publicación de receta, traé esto, verificá aquello, ejecutá esta herramienta, luego producí esta salida."

SQL Connection

A skill is like a view or a report template — it formats and presents information in a consistent, expected way. A workflow is like a stored procedure that coordinates multiple operations, uses branching logic, and has its own error handling.

Conexión SQL

Una habilidad es como una vista o plantilla de reporte — formatea y presenta información de manera consistente. Un workflow es como un procedimiento almacenado que coordina múltiples operaciones, usa lógica de ramificación y tiene su propio manejo de errores.

  3. Improve that workflow. Add or fix three things:
    • A proper Required Inputs section (topic, tone, language, product?)

    • An Expected Output section with a short example of what a good caption looks like
    Mejorá ese workflow. Agregá o corregí tres cosas:
    • Una sección de Entradas Requeridas completa (tema, tono, idioma, producto?)
    • Una sección de Manejo de Errores — ¿qué debería hacer Claude si el tema es demasiado vago para trabajar?
    • Una sección de Resultado Esperado con un ejemplo corto de cómo se ve un buen caption
  4. Create a second workflow: workflows/recipe_post.md
    This workflow takes one of your recipes and turns it into a blog-post-style Instagram caption in Spanish. Think through: what inputs does it need? What steps does Claude follow? What if the recipe name is in English — should it translate it? What's the expected output format?
    Creá un segundo workflow: workflows/recipe_post.md
    Este workflow toma una de tus recetas y la convierte en un caption de Instagram estilo blog en español. Pensá: ¿qué entradas necesita? ¿Qué pasos sigue Claude? ¿Y si el nombre de la receta está en inglés — debería traducirlo? ¿Cuál es el formato de salida esperado?
  5. Self-review test: Read both workflows out loud, slowly. Ask: "If I handed this to a new assistant with zero context, could they follow it without asking a single question?" If the answer is no, keep editing until it is.
    Prueba de autorevision: Leé ambos workflows en voz alta, despacio. Preguntate: "Si le diera esto a un asistente nuevo sin ningún contexto, ¿podría seguirlo sin hacer una sola pregunta?" Si la respuesta es no, seguí editando hasta que lo sea.
