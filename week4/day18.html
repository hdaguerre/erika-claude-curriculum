APIs — Endpoints, Authentication, and the Remote Stored Procedure

Estimated time: 3 hours
ConceptsConceptos

1. What is an API?1. ¿Qué es una API?

An API (Application Programming Interface) is a way to talk to another system over the internet. Instead of opening a website and clicking buttons, you send a structured request to a URL — called an endpoint — and get a structured response back, usually in JSON format.

APIs power everything in the modern software world. When Claude fetches web content, it's using an API. When a tool pulls data from Google Sheets, that's an API. When your phone shows you the weather, it called an API. You've always been surrounded by them — now you understand what they are.

Una API (Interfaz de Programación de Aplicaciones) es una forma de hablar con otro sistema a través de internet. En lugar de abrir un sitio web y hacer clic en botones, enviás una solicitud estructurada a una URL (llamada endpoint) y recibís una respuesta estructurada (generalmente JSON). Las APIs alimentan todo: cuando Claude obtiene contenido web, usa una API. Cuando extraés datos de Google Sheets, eso es una API.

SQL Connection

An API is a remote stored procedure on someone else's server. The endpoint URL = the procedure name on that remote server. You call it, pass it parameters, and get a result set back. You don't see the code inside — you just call it with the right inputs and trust the output.

Conexión SQL

Una API = un procedimiento almacenado remoto. La URL del endpoint = el nombre del procedimiento en un servidor remoto. La clave API = las credenciales de conexión a la base de datos. El cuerpo de la solicitud = los parámetros del procedimiento. La respuesta JSON = el conjunto de resultados. Las cabeceras de autenticación = las credenciales de inicio de sesión en tu cadena de conexión.

2. How API calls work2. Cómo funcionan las llamadas a una API

Every API call has the same four parts:

  • Endpoint — the URL you send the request to, e.g. https://api.anthropic.com/v1/messages
  • Method — GET to fetch data, POST to send data and get a response
  • Headers — metadata sent with every request, including your authentication key
  • Body (optional) — the data you're sending, formatted as JSON

The API responds with a status code (200 = success, 401 = unauthorized, 404 = not found, 429 = rate limited) and a payload — the actual data you asked for.

Cada llamada a una API tiene: un endpoint (la URL a la que enviás), un método (GET para obtener, POST para enviar), cabeceras (metadatos — incluyendo tu clave de autenticación), y a veces un cuerpo (datos que enviás). La API responde con un código de estado (200 = éxito, 404 = no encontrado, 401 = no autorizado) y una carga útil (los datos).

Key Insight — Status Codes

Status codes are how APIs communicate errors. 401 means your key is wrong or missing. 429 means you're calling too fast. 500 means something broke on their end. When a tool fails, check the status code first — it tells you exactly where the problem is.

3. Authentication — the API key3. Autenticación — la clave API

Most APIs require a key — a secret string that proves you're authorized to use them. You include it in every request's headers. Without it, the API returns 401 Unauthorized.

This key lives in a .env file in your project root. Never put it directly in your code. Why? Because code gets shared, committed to git, and accidentally exposed. A .env file stays local.

When a tool needs to call an API, it reads the key from .env using the python-dotenv library — two lines of code that keep your credentials safe.

La mayoría de las APIs requieren una clave — una cadena secreta que demuestra que estás autorizada a usarlas. Esta clave vive en un archivo .env (NUNCA en el código). Cuando un script de herramienta necesita llamar a una API, lee la clave desde .env usando una librería como python-dotenv.

SQL Connection

The API key = the database connection credentials. You don't embed your Oracle password in a SQL script — you store it in a connection configuration and reference it by name. The .env file is exactly that: your credential store. The python-dotenv library = the connection manager that loads those credentials at runtime.

Exercise (~45 to 60 minutes)Ejercicio (~45 a 60 minutos)
  1. You don't need to write code today — you need to understand. Open a browser and search for "Anthropic API documentation" or go to docs.anthropic.com directly.
    Hoy no necesitás escribir código — necesitás entender. Abrí un navegador y buscá "Anthropic API documentation" o andá directamente a docs.anthropic.com.
  2. Find the "Messages" API endpoint. Read the full request structure. Answer these questions before moving on:
    • What fields are required in every request?
    • What is the model field? What values can it take?
    • What is the messages array? What does each message object look like?
    • What does the response object look like? Where is Claude's actual answer in the JSON?
    Encontrá el endpoint de la API "Messages". Leé la estructura completa de la solicitud. Respondé estas preguntas antes de seguir:
    • ¿Qué campos son obligatorios en cada solicitud?
    • ¿Qué es el campo model? ¿Qué valores puede tener?
    • ¿Qué es el array messages? ¿Cómo se ve cada objeto de mensaje?
    • ¿Cómo se ve el objeto de respuesta? ¿Dónde está la respuesta real de Claude en el JSON?
  3. Notice the connection. The messages array in the API is literally what you type in Claude Desktop. You've been "calling the API" conceptually the entire time — just through a graphical interface instead of code.
    Fijate en la conexión. El array messages en la API es literalmente lo que escribís en Claude Desktop. Estuviste "llamando a la API" conceptualmente todo el tiempo — solo a través de una interfaz gráfica en lugar de código.
  4. Optional (if comfortable): Create a .env file in the root of your mi-taller-code project. Add this line:
    ANTHROPIC_API_KEY=your_key_here
    Don't worry about getting a real key yet. This is just about learning the pattern: credentials live in .env, never in code.

    Also read the python-dotenv documentation to see how two lines of code can load that key into any Python script.
    Opcional (si te sentís cómoda): Creá un archivo .env en la raíz de tu proyecto mi-taller-code. Agregá esta línea:
    ANTHROPIC_API_KEY=tu_clave_aquí
    No te preocupés por conseguir una clave real todavía. Esto es solo para aprender el patrón: las credenciales viven en .env, nunca en el código.

    También leé la documentación de python-dotenv para ver cómo dos líneas de código pueden cargar esa clave en cualquier script de Python.
