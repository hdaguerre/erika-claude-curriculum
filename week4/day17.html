Week 4 Day 17

Tools — Python Scripts as Deterministic Execution

Estimated time: 3 hours
ConceptsConceptos

1. What is a tool?1. ¿Qué es una herramienta?

A tool is a Python script in the tools/ folder that does one specific thing reliably. It doesn't think — it executes. Claude decides when to run a tool and what to pass it. The tool just runs the same way every time.

This is the core reliability guarantee: 100% of the time, if you call the tool with the same input, you get the same output. No variation. No "I interpreted this differently today."

Una herramienta es un script de Python en la carpeta tools/ que hace una cosa específica de manera confiable. No piensa — ejecuta. Claude decide cuándo ejecutar una herramienta y qué pasarle. La herramienta simplemente corre de la misma manera cada vez.

Esta es la garantía central de confiabilidad: el 100% de las veces, si llamás la herramienta con la misma entrada, obtenés la misma salida. Sin variación. Sin "hoy lo interpreté diferente."

SQL Connection

A tool is the stored procedure body. Claude is the application calling the procedure. The procedure signature (its input parameters) = the command-line arguments. The result set = the script's output. Just like EXEC save_caption('text', 'filename') does exactly one thing every time — no opinions, no creativity, just execution.

Conexión SQL

Una herramienta = el cuerpo del procedimiento almacenado. Claude = la aplicación que llama al procedimiento. La firma del procedimiento (entradas) = argumentos de línea de comandos. El conjunto de resultados = la salida del script. Igual que un procedimiento almacenado que hace una cosa de manera confiable cuando se llama con los parámetros correctos — sin opiniones, sin creatividad, solo ejecución.

2. Anatomy of a tool2. Anatomía de una herramienta

Every tool script follows the same pattern. It has four parts:

  • Argument parsing — what inputs does the script accept? Uses Python's argparse library
  • Core logic — the actual work: fetch, transform, save, calculate
  • Output — what does it print or return when it's done?
  • Error handling — what happens if an input is missing, a file doesn't exist, a network request fails?

Tools are called from the command line: python tools/save_caption.py --caption "..." --filename "test"

Todo script de herramienta sigue el mismo patrón. Tiene cuatro partes:

  • Análisis de argumentos — ¿qué entradas acepta el script? Usa la librería argparse de Python
  • Lógica central — el trabajo real: obtener datos, transformar, guardar, calcular
  • Salida — ¿qué imprime o devuelve cuando termina?
  • Manejo de errores — ¿qué pasa si falta una entrada, un archivo no existe, una solicitud de red falla?

Las herramientas se llaman desde la línea de comandos: python tools/save_caption.py --caption "..." --filename "test"

Key Insight — Why this matters

Claude might reason slightly differently on different runs — that's how language models work. A Python script never does. For anything that must be consistent (saving files, calling APIs, formatting output, logging data), use a tool. Reserve Claude for what it's good at: reasoning, judgment, and orchestration.

Idea Clave — Por qué importa

Claude puede razonar de manera ligeramente diferente en diferentes ejecuciones — así funcionan los modelos de lenguaje. Un script de Python nunca lo hace. Para cualquier cosa que deba ser consistente (guardar archivos, llamar APIs, formatear salida, registrar datos), usá una herramienta. Reservá a Claude para lo que hace bien: razonar, juzgar y orquestar.

3. Reading a real tool3. Leyendo una herramienta real

Before you write your own tool, you're going to read two real ones. They live in the skill-creator/scripts/ folder inside the main erika project. Reading existing scripts is the fastest way to internalize the pattern. You're not memorizing syntax — you're understanding structure.

As you read, ask for each function: What does it accept? What does it do? What does it return? That's all you need.

Antes de escribir tu propia herramienta, vas a leer dos reales. Están en la carpeta skill-creator/scripts/ dentro del proyecto erika principal. Leer scripts existentes es la manera más rápida de internalizar el patrón. No estás memorizando sintaxis — estás entendiendo la estructura.

Mientras leés, preguntate por cada función: ¿Qué acepta? ¿Qué hace? ¿Qué devuelve? Eso es todo lo que necesitás.

SQL Connection

Reading utils.py is like reading a package of shared utility procedures in Oracle — procedures other procedures call. quick_validate.py is like a pre-execution validation check: it confirms the inputs are sane before the main logic runs. Both patterns will feel very familiar.

Conexión SQL

Leer utils.py es como leer un paquete de procedimientos utilitarios compartidos en Oracle — procedimientos que otros procedimientos llaman. quick_validate.py es como una verificación de validación antes de la ejecución: confirma que las entradas son correctas antes de que corra la lógica principal. Los dos patrones te van a resultar muy familiares.

1 What is the key difference between a tool and an agent? ¿Cuál es la diferencia clave entre una herramienta y un agente?
Tools execute deterministically — same input always produces the same output, every single time.
Agents reason and adapt — they interpret, make decisions, and their output can vary based on context. Claude is an agent. save_caption.py is a tool.
Las herramientas ejecutan de manera determinista — la misma entrada siempre produce la misma salida, cada vez.
Los agentes razonan y se adaptan — interpretan, toman decisiones, y su salida puede variar según el contexto. Claude es un agente. save_caption.py es una herramienta.
2 Why would you use a Python tool instead of just asking Claude to save the file? ¿Por qué usarías una herramienta de Python en vez de simplemente pedirle a Claude que guarde el archivo?
Reliability. For anything that must work consistently every time — saving files, calling APIs, formatting data, logging — a deterministic script is better than AI reasoning. Claude might save the file differently on a different day, in a different format, to a different location. A script will not.
Confiabilidad. Para cualquier cosa que deba funcionar de manera consistente cada vez — guardar archivos, llamar APIs, formatear datos, registrar información — un script determinista es mejor que el razonamiento de IA. Claude podría guardar el archivo de manera diferente otro día, en otro formato, en otra ubicación. Un script no.
3 What is the SQL analogy for a tool? ¿Cuál es la analogía SQL para una herramienta?
The stored procedure body — called with input parameters, executes the same logic every time, returns the same type of output. The command-line arguments = the procedure parameters. Claude = the application calling EXEC.
El cuerpo del procedimiento almacenado — se llama con parámetros de entrada, ejecuta la misma lógica cada vez, devuelve el mismo tipo de salida. Los argumentos de línea de comandos = los parámetros del procedimiento. Claude = la aplicación que llama EXEC.
