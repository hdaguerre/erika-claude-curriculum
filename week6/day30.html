2. Integration verification2. Verificación de integración

Running a pipeline end-to-end always surfaces integration bugs — problems that don't appear when you test each step in isolation. The most common ones:

  • Column name mismatch: Step 1 outputs company_name, Step 2 expects company — KeyError.
  • Type mismatch: Step 1 outputs a number as a string, Step 2 tries to compare it as an integer.
  • Missing field: Some rows have no domain — the tech stack function receives None and crashes.

After each step, do a quick check: column count, null rate in key columns, a sample of 3–5 values. This takes 30 seconds and catches most problems before they cascade.

Ejecutar un pipeline de punta a punta siempre revela bugs de integración — problemas que no aparecen cuando probás cada paso de forma aislada. Los más comunes:

  • Nombre de columna incorrecto: El Paso 1 produce company_name, el Paso 2 espera company — KeyError.
  • Tipo incorrecto: El Paso 1 produce un número como string, el Paso 2 intenta compararlo como entero.
  • Campo faltante: Algunas filas no tienen dominio — la función de tech stack recibe None y crashea.

Después de cada paso, hacé una verificación rápida: conteo de columnas, tasa de nulos en columnas clave, una muestra de 3–5 valores. Esto toma 30 segundos y atrapa la mayoría de los problemas antes de que se propaguen.

Quick CheckVerificación Rápida
1 What is an integration bug? ¿Qué es un bug de integración?
When the output format of one pipeline step doesn't match what the next step expects — a wrong column name, a missing field, or a type mismatch (e.g., string where a number is expected). Each step works fine in isolation, but they break when connected.
Cuando el formato de output de un paso del pipeline no coincide con lo que espera el siguiente paso — un nombre de columna incorrecto, un campo faltante o un tipo incorrecto (por ejemplo, un string donde se espera un número). Cada paso funciona bien de forma aislada, pero se rompen cuando se conectan.
2 What columns does the enriched CSV have after Week 6? ¿Qué columnas tiene el CSV enriquecido después de la Semana 6?
first_name, last_name, email, company, domain, company_size, tech_stack, hiring_signal. Eight columns — a complete enriched lead record ready for scoring or CRM push.
first_name, last_name, email, company, domain, company_size, tech_stack, hiring_signal. Ocho columnas — un registro de lead enriquecido completo, listo para puntuar o empujar al CRM.
3 Why run on a small sample first? ¿Por qué ejecutar en una muestra pequeña primero?
To catch bugs cheaply before spending API credits on thousands of rows. Each Apify run costs credits — a bug on row 1 that you discover at row 500 means you wasted credits on 499 bad runs. Test small, fix issues, then scale up.
Para atrapar bugs de forma barata antes de gastar créditos de API en miles de filas. Cada ejecución de Apify cuesta créditos — un bug en la fila 1 que descubrís en la fila 500 significa que desperdiciaste créditos en 499 ejecuciones malas. Probá en pequeño, arreglá los problemas, luego escalá.
