Apify & the Actor Model

Estimated time: ~3 hours
ConceptsConceptos

1. What Apify is and why it exists1. Qué es Apify y por qué existe

The web has no public API for most data. Apify solves this: it hosts pre-built scrapers called Actors that run headless browsers, extract data from any website, and return structured JSON. Think of it as a library of stored procedures written by strangers — you call them with inputs, they return a result set.

The Apify console at console.apify.com lets you browse thousands of Actors, test them with a UI, see their input schema and output format, and run them manually before ever writing code. It is the catalog for this library of scrapers.

La web no tiene una API pública para la mayoría de los datos. Apify resuelve esto: aloja scrapers pre-construidos llamados Actors que ejecutan navegadores headless, extraen datos de cualquier sitio web y devuelven JSON estructurado. Pensalo como una biblioteca de stored procedures escritos por otros — los llamás con inputs y te devuelven un result set.

La consola de Apify en console.apify.com te permite navegar miles de Actors, probarlos con una interfaz gráfica, ver su esquema de input y formato de output, y ejecutarlos manualmente antes de escribir una sola línea de código.

SQL Connection

An Apify Actor is a vendor-hosted stored procedure. You pass it parameters (a list of URLs or search terms), it runs on Apify's infrastructure, and returns results in JSON. You don't maintain the scraper — just the call. Same pattern as calling a package owned by another team: you know the signature, not the internals.

Conexión SQL

Un Apify Actor es un stored procedure hosteado por un tercero. Le pasás parámetros (una lista de URLs o términos de búsqueda), corre en la infraestructura de Apify, y devuelve resultados en JSON. No mantenés el scraper — solo la llamada. El mismo patrón que llamar a un package de otro equipo: conocés la firma, no los internos.

2. The Built With Actor — your first real enrichment signal2. El Actor Built With — tu primera señal de enriquecimiento real

BuiltWith tells you what technology stack a website is running: CRM, marketing tools, analytics, e-commerce platform. A company running HubSpot + Webflow is different from one running Salesforce + Marketo — these are ICP signals.

You search for the "Built With" or "tech stack checker" Actor in the Apify console, run it on a domain, and get back a JSON object listing every technology category detected. This becomes your tech_stack enrichment column.

BuiltWith te dice qué stack tecnológico usa un sitio web: CRM, herramientas de marketing, analytics, plataforma de e-commerce. Una empresa que usa HubSpot + Webflow es diferente a una que usa Salesforce + Marketo — estas son señales de ICP.

Buscás el Actor "Built With" o "tech stack checker" en la consola de Apify, lo ejecutás en un dominio, y obtenés un objeto JSON con todas las categorías de tecnología detectadas. Esto se convierte en tu columna de enriquecimiento tech_stack.

Key Insight

Knowing someone's tech stack before you reach out is like knowing their database schema before you query it — it tells you what they care about and what they're probably missing.

Idea Clave

Saber el stack tecnológico de alguien antes de contactarlo es como conocer su esquema de base de datos antes de consultarla — te dice qué les importa y qué probablemente les falta.

Exercise (~70 minutes)Ejercicio (~70 minutos)
  1. Go to apify.com and create a free account. Confirm your email address when prompted.
    Andá a apify.com y creá una cuenta gratuita. Confirmá tu dirección de email cuando te lo pidan.
  2. In the Apify console, use the search bar to find the "Built With" or "tech stack checker" Actor. Click into its page and read its input schema and output format before running it.
    En la consola de Apify, usá la barra de búsqueda para encontrar el Actor "Built With" o "tech stack checker". Entrá a su página y leé su esquema de input y formato de output antes de ejecutarlo.
  3. Run the Actor manually on 3 target domains you choose. Pick B2B companies — for example: hubspot.com, drift.com, gong.io. Click "Start" and wait for it to finish.
    Ejecutá el Actor manualmente en 3 dominios objetivo que elijas. Elegí empresas B2B — por ejemplo: hubspot.com, drift.com, gong.io. Hacé clic en "Start" y esperá que termine.
  4. Look at the JSON output for each domain. Identify the technology categories — CRM, analytics, advertising, e-commerce, CDN. Notice which fields are most consistent across runs.
    Mirá el output JSON de cada dominio. Identificá las categorías de tecnología — CRM, analytics, publicidad, e-commerce, CDN. Fijate qué campos son más consistentes entre ejecuciones.
  5. Ask Claude: "What does this JSON output tell me about this company from a GTM perspective? What signals would I look for?" Paste in one of your outputs. Read Claude's analysis.
    Preguntale a Claude: "¿Qué me dice este output JSON sobre esta empresa desde una perspectiva GTM? ¿Qué señales buscaría?" Pegá uno de tus outputs. Leé el análisis de Claude.
  6. Explore 2 other Actors in the Apify console — any category interests you. Note their names and what data they return. You're building a mental catalog of available signals.
    Explorá 2 Actors más en la consola de Apify — cualquier categoría que te interese. Anotá sus nombres y qué datos devuelven. Estás construyendo un catálogo mental de señales disponibles.
