Apollo API Basics

Estimated time: ~3.5 hours
ConceptsConceptos

1. What Apollo is and what its API does1. Qué es Apollo y qué hace su API

Apollo.io is a B2B data and outreach platform — think of it as a database of ~275 million contacts combined with email sequencing tools. GTM teams use it to find prospects, enrich data, and run outbound campaigns all in one place.

The Apollo API lets you push contacts into the system programmatically. Instead of importing a CSV manually through the UI, your Python script sends each enriched lead directly to Apollo via an HTTP POST — no human clicks required. This is how a proper pipeline works: clean → enrich → push to CRM/outreach tool, all automated.

Apollo.io es una plataforma de datos B2B y outreach — pensalo como una base de datos de ~275 millones de contactos combinada con herramientas de secuenciación de emails. Los equipos GTM lo usan para encontrar prospectos, enriquecer datos y ejecutar campañas de outbound todo en un solo lugar.

La API de Apollo te permite empujar contactos al sistema de forma programática. En vez de importar un CSV manualmente a través de la UI, tu script Python envía cada lead enriquecido directamente a Apollo via HTTP POST — sin clicks humanos requeridos. Así es como funciona un pipeline correcto: limpiar → enriquecer → empujar al CRM/herramienta de outreach, todo automatizado.

SQL Connection

Apollo's contact database is a remote table. Your push_contact(row) function is equivalent to:
MERGE INTO apollo.contacts USING (:email) ON (email = :email)
WHEN NOT MATCHED THEN INSERT (first_name, last_name, email, organization_name, website_url)
VALUES (:first_name, :last_name, :email, :org, :url);
If the contact exists, Apollo may update it. If not, it inserts — classic MERGE behavior.

Conexión SQL

La base de datos de contactos de Apollo es una tabla remota. Tu función push_contact(row) es equivalente a:

WHEN NOT MATCHED THEN INSERT (first_name, last_name, email, organization_name, website_url)

Si el contacto existe, Apollo puede actualizarlo. Si no, inserta — comportamiento clásico de MERGE.

Key Insight

Key endpoint: POST https://api.apollo.io/v1/contacts with the header X-Api-Key: your_key and a JSON body containing the contact fields. Apollo returns a contact ID if the record was created successfully.

Idea Clave

Endpoint clave: POST https://api.apollo.io/v1/contacts con el header X-Api-Key: tu_clave y un cuerpo JSON con los campos del contacto. Apollo retorna un contact ID si el registro fue creado exitosamente.

2. Apollo API auth + key endpoints2. Auth de la API de Apollo + endpoints clave

Authentication: Apollo uses an API key passed as a header: X-Api-Key: your_key. Unlike some APIs that use Bearer tokens, Apollo's key goes directly in the request headers.

Required input fields: first_name, last_name, email, organization_name, website_url. These map directly to columns in your enriched CSV — which is exactly why you built that CSV the way you did.

Rate limits: Apollo's free tier is strict. The safest approach is to add a small delay between API calls (1–2 seconds) and never run the script on thousands of rows without upgrading to a paid plan. Always test with mocks first, then with a small batch of real data.

Autenticación: Apollo usa una API key pasada como header: X-Api-Key: tu_clave. A diferencia de algunas APIs que usan tokens Bearer, la clave de Apollo va directamente en los headers de la petición.

Campos de input requeridos: first_name, last_name, email, organization_name, website_url. Estos mapean directamente a columnas en tu CSV enriquecido — que es exactamente por qué construiste ese CSV de esa manera.

Rate limits: El tier gratuito de Apollo es estricto. El enfoque más seguro es agregar un pequeño delay entre llamadas a la API (1–2 segundos) y nunca ejecutar el script en miles de filas sin actualizar a un plan pago. Siempre probá con mocks primero, luego con un lote pequeño de datos reales.

SQL Connection

Rate limiting is like bulk insert batching. When you INSERT thousands of Oracle rows at once, you batch them and commit periodically to avoid log overflow. Same principle: send to Apollo in small batches, pause between them, respect the system's limits.

Conexión SQL

El rate limiting es como el batching de inserts masivos. Cuando insertás miles de filas en Oracle de golpe, las agrupás en batches y hacés commit periódicamente para evitar overflow del log. Mismo principio: enviar a Apollo en batches pequeños, pausar entre ellos, respetar los límites del sistema.
