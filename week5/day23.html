1. What HTTP requests are1. Qué son las peticiones HTTP

Every time a website loads data or an API returns information, an HTTP request is happening in the background. It's a standardized way for one computer to ask another computer for data — or to send it data and get a result back.

The two request types you'll use constantly:

  • GET — retrieve data. You're asking for something. No data sent, just a request. Think: SELECT query.
  • POST — send data and get a result back. You provide input parameters and the API processes them. Think: calling a stored procedure with parameters.

Headers are metadata you attach to the request — your API key, what format you want back, your identity. Think of them as the connection metadata in a tnsnames.ora file — but sent with every call.

JSON is the standard response format. It looks like a Python dictionary: {"name": "Acme", "size": 200}. When an API returns data, it almost always comes back as JSON.

Cada vez que un sitio web carga datos o una API devuelve información, hay una petición HTTP ocurriendo en segundo plano. Es una forma estandarizada para que una computadora le pida datos a otra — o le envíe datos y obtenga un resultado.

Los dos tipos de petición que vas a usar constantemente:

  • GET — recuperar datos. Estás pidiendo algo. No se envían datos, solo una petición. Pensalo como: una consulta SELECT.
  • POST — enviar datos y obtener un resultado. Proporcionás parámetros de entrada y la API los procesa. Pensalo como: llamar a un stored procedure con parámetros.

Los headers son metadatos que adjuntás a la petición — tu API key, el formato que querés recibir, tu identidad. Pensalos como los metadatos de conexión en un archivo tnsnames.ora — pero enviados con cada llamada.

JSON es el formato de respuesta estándar. Se parece a un diccionario Python: {"name": "Acme", "size": 200}. Cuando una API devuelve datos, casi siempre vienen como JSON.

SQL Connection

An API endpoint is like an Oracle DBLINK — you're reaching out to a remote database you don't own and asking it to run a specific query and return the result. You can't see their tables or schema. You just call the endpoint (like calling a remote procedure) and get JSON back (like getting a result set). GET = SELECT. POST = execute stored procedure.

Conexión SQL

Un endpoint de API es como un DBLINK de Oracle — estás llegando a una base de datos remota que no es tuya y pidiéndole que ejecute una consulta específica y devuelva el resultado. No podés ver sus tablas ni su esquema. Solo llamás al endpoint (como llamar a un procedimiento remoto) y recibís JSON (como recibir un conjunto de resultados). GET = SELECT. POST = ejecutar stored procedure.

2. The requests library in Python2. La librería requests en Python

The requests library is the standard Python tool for making HTTP calls. The pattern is always the same:

import requests

response = requests.get("https://api.example.com/data")
data = response.json()

Three lines: import, call, parse. That's it. The response object also has a .status_code property that tells you whether the call succeeded:

  • 200 — OK. Everything worked.
  • 401 — Unauthorized. Your credentials are wrong or missing.
  • 404 — Not found. The endpoint or resource doesn't exist.
  • 429 — Too many requests. You've hit the API's rate limit. Slow down.

La librería requests es la herramienta estándar de Python para hacer llamadas HTTP. El patrón es siempre el mismo:

import requests

response = requests.get("https://api.example.com/data")
data = response.json()

Tres líneas: importar, llamar, parsear. Eso es todo. El objeto response también tiene una propiedad .status_code que te dice si la llamada tuvo éxito:

  • 200 — OK. Todo funcionó.
  • 401 — No autorizado. Tus credenciales son incorrectas o faltan.
  • 404 — No encontrado. El endpoint o recurso no existe.
  • 429 — Demasiadas peticiones. Alcanzaste el límite de tasa de la API. Frenate.
Key Insight

When you see a 429 error, the API is telling you you're calling it too fast. Don't retry immediately — add a time.sleep(1) between calls. APIs have rate limits to prevent abuse, and if you hit them too aggressively, you can get your key temporarily or permanently blocked.

Idea Clave

Cuando veas un error 429, la API te está diciendo que la estás llamando demasiado rápido. No reintentes inmediatamente — agregá un time.sleep(1) entre llamadas. Las APIs tienen límites de tasa para prevenir abuso, y si los golpeás demasiado agresivamente, pueden bloquear tu clave temporalmente o permanentemente.
