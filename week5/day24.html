1. What enrichment means in GTM work1. Qué significa enriquecimiento en el trabajo GTM

You start with a CSV of names and domains. Enrichment means making API calls to add more columns — company size, tech stack, hiring signals, revenue estimates, industry. Each API call "enriches" one row with data you didn't have before.

The result is a more complete picture of each lead: instead of just a name and domain, you have enough context to decide whether to reach out, how to personalize the message, and what offer makes sense.

The enrichment loop:

  1. Load the cleaned CSV into a DataFrame
  2. For each row, make an API call using the domain as the lookup key
  3. Parse the JSON response and extract the fields you want
  4. Write the result back to a new column in that row
  5. Move to the next row
  6. Export the enriched CSV when done

Comenzás con un CSV de nombres y dominios. Enriquecimiento significa hacer llamadas a APIs para agregar más columnas — tamaño de empresa, stack tecnológico, señales de contratación, estimaciones de ingresos, industria. Cada llamada a la API "enriquece" una fila con datos que antes no tenías.

El resultado es una imagen más completa de cada lead: en lugar de solo un nombre y un dominio, tenés suficiente contexto para decidir si contactar, cómo personalizar el mensaje, y qué oferta tiene sentido.

El bucle de enriquecimiento:

  1. Cargar el CSV limpio en un DataFrame
  2. Para cada fila, hacer una llamada a la API usando el dominio como clave de búsqueda
  3. Parsear la respuesta JSON y extraer los campos que querés
  4. Escribir el resultado de vuelta a una nueva columna en esa fila
  5. Pasar a la siguiente fila
  6. Exportar el CSV enriquecido cuando terminés
SQL Connection

Enrichment is a JOIN against a remote API instead of a local table. The concept is identical to: UPDATE leads SET company_size = api_result WHERE domain = lead.domain — except you're executing this row by row using a Python for loop, calling an external HTTP endpoint instead of querying a local table. The API is your remote data source.

Conexión SQL

El enriquecimiento es un JOIN contra una API remota en lugar de una tabla local. El concepto es idéntico a: UPDATE leads SET company_size = api_result WHERE domain = lead.domain — excepto que lo estás ejecutando fila por fila usando un bucle for en Python, llamando a un endpoint HTTP externo en lugar de consultar una tabla local. La API es tu fuente de datos remota.

2. Building tools/enrich_list.py2. Construyendo tools/enrich_list.py

The structure of every enrichment script follows the same pattern:

  1. Load the cleaned CSV: df = pd.read_csv("tools/cleaned_leads.csv")
  2. Loop through each row: for index, row in df.iterrows():
  3. Make the API call using the domain: response = requests.get(url + row["domain"], headers=headers)
  4. Extract the field from JSON: company_size = response.json().get("metrics", {}).get("employees")
  5. Write back: df.at[index, "company_size"] = company_size

After the loop, export: df.to_csv("tools/enriched_leads.csv", index=False)

1 In lead enrichment, what is the equivalent of a SQL JOIN? En el enriquecimiento de leads, ¿cuál es el equivalente de un SQL JOIN?
The API call — you're joining against a remote table via HTTP instead of a local one. The domain is the JOIN key, the API response is the remote "table" being joined, and the fields you extract become new columns in your result set.
La llamada a la API — estás haciendo un join contra una tabla remota via HTTP en lugar de una local. El dominio es la clave del JOIN, la respuesta de la API es la "tabla" remota que se une, y los campos que extraés se convierten en nuevas columnas en tu conjunto de resultados.
2 What's the main risk when calling an API inside a for loop with many rows? ¿Cuál es el principal riesgo al llamar a una API dentro de un bucle for con muchas filas?
Rate limiting — hitting the API too many times too fast causes 429 errors. The API throttles you for exceeding its call limits. Add a time.sleep(0.5) between calls to stay under the rate limit, and add error handling for when it happens anyway (tomorrow's lesson).
El límite de tasa — llamar a la API demasiadas veces demasiado rápido causa errores 429. La API te limita por exceder sus límites de llamadas. Agregá un time.sleep(0.5) entre llamadas para mantenerte bajo el límite de tasa, y agregá manejo de errores para cuando suceda igual (lección de mañana).
