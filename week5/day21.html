1. Python syntax crash course for SQL developers1. Curso intensivo de sintaxis Python para desarrolladores SQL

You already know how to think like a programmer — you've been doing it in Oracle for years. Python just uses different keywords. Here's the direct translation:

  • Variables = column values you store temporarily. name = "Erika" is the same as DECLARE @name VARCHAR := 'Erika' in PL/SQL. You name a slot, put a value in it, use it later.
  • Lists = result sets. leads = ["company1", "company2", "company3"] is like a single-column SELECT result — an ordered set of values you can loop through.
  • Dictionaries = rows. lead = {"name": "Acme", "domain": "acme.com"} is exactly a row with named columns. lead["name"] is like referencing a column by name in a SELECT.
  • For loops = CURSOR FOR LOOP. for row in rows: is the same pattern as FOR rec IN cursor LOOP in PL/SQL — step through each item, do something with it, move to the next.
  • Functions = stored procedures you call by name. def clean_email(email): defines a reusable block of logic, just like a stored procedure — pass in a value, get a result back.

Ya sabés pensar como programadora — lo estuviste haciendo en Oracle durante años. Python solo usa palabras clave diferentes. Acá está la traducción directa:

  • Variables = valores de columna que guardás temporalmente. name = "Erika" es lo mismo que DECLARE @name VARCHAR := 'Erika' en PL/SQL. Nombrás un espacio, le ponés un valor, lo usás después.
  • Listas = conjuntos de resultados. leads = ["company1", "company2", "company3"] es como el resultado de un SELECT de una sola columna — un conjunto ordenado de valores por los que podés iterar.
  • Diccionarios = filas. lead = {"name": "Acme", "domain": "acme.com"} es exactamente una fila con columnas nombradas. lead["name"] es como referenciar una columna por nombre en un SELECT.
  • Bucles for = CURSOR FOR LOOP. for row in rows: es el mismo patrón que FOR rec IN cursor LOOP en PL/SQL — pasás por cada ítem, hacés algo con él, seguís al próximo.
  • Funciones = stored procedures que llamás por nombre. def clean_email(email): define un bloque de lógica reutilizable, igual que un stored procedure — pasás un valor, recibís un resultado.

2. How to run a Python file from the terminal2. Cómo ejecutar un archivo Python desde la terminal

Running a Python script is one command: python filename.py. Open your terminal, navigate to the folder where the file lives, and type it. The script runs top to bottom, just like a PL/SQL anonymous block executing in SQL*Plus.

If your system has both Python 2 and Python 3, you may need to use python3 filename.py. Run python --version first to see which you have.

Ejecutar un script Python es un solo comando: python filename.py. Abrí tu terminal, navegá a la carpeta donde está el archivo, y escribilo. El script corre de arriba a abajo, igual que un bloque anónimo PL/SQL ejecutando en SQL*Plus.

Si tu sistema tiene Python 2 y Python 3, puede que necesites usar python3 filename.py. Primero ejecutá python --version para ver cuál tenés.

1 What is the Python equivalent of a SQL cursor FOR loop? ¿Cuál es el equivalente en Python de un cursor FOR LOOP de SQL?
A for loop: for item in list: — it steps through every element in a list exactly the way FOR rec IN cursor LOOP steps through every row in a cursor result.
Un bucle for: for item in list: — pasa por cada elemento de una lista exactamente de la misma manera que FOR rec IN cursor LOOP pasa por cada fila en el resultado de un cursor.
2 What Python data structure is most like a SQL row with named columns? ¿Qué estructura de datos de Python se parece más a una fila SQL con columnas nombradas?
A dictionary — {"name": "Acme", "domain": "acme.com"}. Each key is a column name, each value is the column value for that row. You access values with row["column_name"], just like referencing a column in SQL.
Un diccionario — {"name": "Acme", "domain": "acme.com"}. Cada clave es un nombre de columna, cada valor es el valor de esa columna para esa fila. Accedés a los valores con row["column_name"], igual que referenciar una columna en SQL.
