ICP Scoring Model

Estimated time: ~3.5 hours
ConceptsConceptos

1. What an ICP scoring model is1. ¿Qué es un modelo de puntuación ICP?

ICP = Ideal Customer Profile. A scoring model translates "good fit" criteria into numbers — each lead gets a score based on how closely it matches your ICP. Higher score = higher priority outreach.

The model works by assigning weights to signals. For example:

  • Company uses HubSpot (tech stack match) = +20 points
  • Company is hiring RevOps (hiring signal) = +15 points
  • Company size 50–500 employees = +10 points
  • Company is in the US = +5 points

Total score out of 100. The highest-scored leads are who you reach out to first.

ICP = Ideal Customer Profile (Perfil de Cliente Ideal). Un modelo de puntuación traduce los criterios de "buen ajuste" en números — cada lead recibe una puntuación basada en qué tan bien coincide con tu ICP. Puntuación más alta = prioridad más alta para el outreach.

El modelo funciona asignando pesos a señales. Por ejemplo:

  • La empresa usa HubSpot (coincidencia de tech stack) = +20 puntos
  • La empresa está contratando RevOps (señal de contratación) = +15 puntos
  • Tamaño de empresa 50–500 empleados = +10 puntos
  • La empresa está en EE. UU. = +5 puntos

Puntuación total sobre 100. Los leads con mayor puntuación son a quienes primero contactás.

SQL Connection

A scoring model is a computed column query. You've written these before in Oracle:

SELECT *,
  (CASE WHEN tech_stack LIKE '%HubSpot%' THEN 20 ELSE 0 END
   + CASE WHEN hiring_signal = TRUE THEN 15 ELSE 0 END
   + CASE WHEN employee_count BETWEEN 50 AND 500 THEN 10 ELSE 0 END
   + CASE WHEN country = 'US' THEN 5 ELSE 0 END) AS score
FROM leads
ORDER BY score DESC

The Python version does the exact same logic — it just runs row by row on a CSV instead of on a database table.

Conexión SQL

Un modelo de puntuación es una consulta de columna calculada. Ya escribiste estas en Oracle:

SELECT *,
  (CASE WHEN tech_stack LIKE '%HubSpot%' THEN 20 ELSE 0 END
   + CASE WHEN hiring_signal = TRUE THEN 15 ELSE 0 END
   + CASE WHEN employee_count BETWEEN 50 AND 500 THEN 10 ELSE 0 END
   + CASE WHEN country = 'US' THEN 5 ELSE 0 END) AS score
FROM leads
ORDER BY score DESC

La versión en Python hace exactamente la misma lógica — solo que corre fila por fila sobre un CSV en vez de sobre una tabla de base de datos.

2. Building tools/score_leads.py2. Construyendo tools/score_leads.py

This script takes the enriched CSV from Week 6 and adds a score column.

  • Input: tools/enriched_sample.csv — the file you built during enrichment week
  • Logic: A series of if/else checks per row, adding weighted points for each signal that matches
  • Output: tools/scored_leads.csv — same data, new score column, sorted highest to lowest

Design principle: Start with 5 simple criteria. Get feedback from Hernan. Adjust weights. Don't over-engineer on the first pass — a simple model you can tune is better than a complex model you can't explain.

Este script toma el CSV enriquecido de la Semana 6 y agrega una columna score.

  • Entrada: tools/enriched_sample.csv — el archivo que construiste durante la semana de enriquecimiento
  • Lógica: Una serie de verificaciones if/else por fila, sumando puntos ponderados por cada señal que coincide
  • Salida: tools/scored_leads.csv — mismos datos, nueva columna score, ordenados de mayor a menor

Principio de diseño: Empezá con 5 criterios simples. Pedile feedback a Hernan. Ajustá los pesos. No sobre-ingenierees en la primera iteración — un modelo simple que podés ajustar es mejor que uno complejo que no podés explicar.

Key Insight

A scoring model is just a decision rule made explicit. Every salesperson already has one in their head — "I look for companies that use X, are this size, hire for Y role." The model makes that instinct repeatable and measurable.

Idea Clave

Un modelo de puntuación es simplemente una regla de decisión hecha explícita. Todo vendedor ya tiene una en su cabeza — "busco empresas que usan X, tienen este tamaño, contratan para el rol Y." El modelo hace que ese instinto sea repetible y medible.

Quick CheckVerificación Rápida
1 What does an ICP scoring model produce? ¿Qué produce un modelo de puntuación ICP?
A score for each lead based on how closely they match your Ideal Customer Profile — allows you to prioritize outreach by fit. The higher the score, the more that company looks like your best customers.
Una puntuación para cada lead basada en qué tan bien coinciden con tu Perfil de Cliente Ideal — te permite priorizar el outreach por ajuste. Cuanto más alta la puntuación, más se parece esa empresa a tus mejores clientes.
2 What is the SQL equivalent of a scoring model? ¿Cuál es el equivalente SQL de un modelo de puntuación?
A computed column query with CASE WHEN statements for each scoring criterion, summed into a total score column. You've written exactly this kind of query in Oracle — it's the same pattern, just running on a CSV instead of a database table.
Una consulta de columna calculada con sentencias CASE WHEN para cada criterio de puntuación, sumadas en una columna de puntuación total. Ya escribiste exactamente este tipo de consulta en Oracle — es el mismo patrón, solo corriendo sobre un CSV en vez de una tabla de base de datos.
3 Why start with only 5 scoring criteria? ¿Por qué empezar con solo 5 criterios de puntuación?
To keep it simple, testable, and easy to tune. You can always add more criteria after validating the first version with real output. A model with 5 well-chosen signals beats a model with 20 signals you don't understand.
Para mantenerlo simple, testeable y fácil de ajustar. Siempre podés agregar más criterios después de validar la primera versión con resultados reales. Un modelo con 5 señales bien elegidas supera a uno con 20 señales que no entendés.
