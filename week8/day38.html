1. Pushing contacts to Apollo programmatically1. Enviando contactos a Apollo de forma programática

Apollo's Contacts API lets you create or update contact records without touching the UI. You send a JSON payload with the contact's details and Apollo creates the record if it doesn't exist, or updates it if it does.

The key endpoint:

POST https://api.apollo.io/v1/contacts

Headers:

{"X-Api-Key": "your_key", "Content-Type": "application/json"}

Body (minimum required fields):

{
  "first_name": "...",
  "last_name": "...",
  "email": "...",
  "organization_name": "...",
  "website_url": "..."
}

Apollo returns a JSON response with the created contact's ID. Log this — you'll need it if you ever want to update the record later.

La API de Contactos de Apollo te permite crear o actualizar registros de contactos sin tocar la interfaz. Enviás un payload JSON con los detalles del contacto y Apollo crea el registro si no existe, o lo actualiza si existe.

El endpoint clave:

POST https://api.apollo.io/v1/contacts

Headers:

{"X-Api-Key": "tu_clave", "Content-Type": "application/json"}

Body (campos mínimos requeridos):

{
  "first_name": "...",
  "last_name": "...",
  "email": "...",
  "organization_name": "...",
  "website_url": "..."
}

Apollo devuelve una respuesta JSON con el ID del contacto creado. Registralo — lo vas a necesitar si querés actualizar el registro después.

SQL Connection

This is a remote MERGE INTO statement — the same upsert pattern you know from Oracle. Apollo's API handles the "does this record exist?" check for you. Think of each API call as executing:

MERGE INTO apollo.contacts c
USING (SELECT :email FROM dual) src
ON (c.email = src.email)
WHEN MATCHED THEN UPDATE SET ...
WHEN NOT MATCHED THEN INSERT (...) VALUES (...)

2. Rate limits and batching2. Límites de velocidad y procesamiento por lotes

Apollo's free tier limits how many API calls you can make per minute. If you loop through all rows without any delay, you'll hit a 429 Too Many Requests error and the script will crash partway through.

The safe pattern:

  • Process in batches of 10 rows
  • Add time.sleep(1) between batches — one second pause is usually enough
  • Log each push: print the company name, the HTTP status code, and "success" or the error message
  • If a push fails: log the error, continue to the next row — don't let one failure stop the whole pipeline

El nivel gratuito de Apollo limita cuántas llamadas a la API podés hacer por minuto. Si iterás sobre todas las filas sin ninguna demora, vas a llegar al error 429 Too Many Requests y el script va a fallar a mitad de camino.

El patrón seguro:

  • Procesar en lotes de 10 filas
  • Agregar time.sleep(1) entre lotes — una pausa de un segundo suele ser suficiente
  • Registrar cada push: imprimir el nombre de la empresa, el código de estado HTTP y "success" o el mensaje de error
  • Si un push falla: registrar el error, continuar con la siguiente fila — no dejar que un fallo detenga todo el pipeline
Key Insight

The score threshold (score >= 60) is your filter before pushing. You're not pushing everything — you're pushing qualified leads. This keeps Apollo clean and makes sure your outreach sequences start with the right companies, not your entire list.

Idea Clave

El umbral de puntuación (score >= 60) es tu filtro antes de enviar. No enviás todo — enviás leads calificados. Esto mantiene Apollo limpio y asegura que tus secuencias de outreach empiecen con las empresas correctas, no con toda tu lista.
