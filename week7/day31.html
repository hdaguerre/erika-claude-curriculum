ConceptsConceptos

1. What an MCP server is1. Qué es un servidor MCP

MCP (Model Context Protocol) is a standard that lets Claude connect to external tools and data sources. An MCP server exposes capabilities — read a file, query an API, search a database — that Claude can call mid-conversation. It's what transforms Claude from a chat assistant into an active participant in your system.

Without MCP: Claude reads text you paste in. With MCP: Claude can directly call Figma, read your files, query APIs — all without you copy-pasting anything.

MCP (Model Context Protocol) es un estándar que le permite a Claude conectarse a herramientas y fuentes de datos externas. Un servidor MCP expone capacidades — leer un archivo, consultar una API, buscar en una base de datos — que Claude puede invocar en medio de una conversación. Es lo que transforma a Claude de un asistente de chat a un participante activo en tu sistema.

Sin MCP: Claude lee el texto que le pegás. Con MCP: Claude puede llamar directamente a Figma, leer tus archivos, consultar APIs — todo sin que vos tengas que copiar ni pegar nada.

SQL Connection

An MCP server is like an Oracle Database Link combined with a package of stored procedures. The DBLINK gives Claude access to the remote system. The package procedures define what operations are allowed. Claude doesn't own the data — it just gets authorized read/write access through the link.

Conexión SQL

Un servidor MCP es como un Database Link de Oracle combinado con un paquete de stored procedures. El DBLINK le da a Claude acceso al sistema remoto. Los procedimientos del paquete definen qué operaciones están permitidas. Claude no es dueño de los datos — solo obtiene acceso autorizado de lectura/escritura a través del enlace.

2. MCP in the WAT framework2. MCP en el framework WAT

In the WAT framework, MCP servers are Tools — but richer than plain Python scripts. Instead of calling python tools/my_script.py, Claude calls an MCP endpoint directly.

The Figma MCP server, for example, lets Claude call get_design_context(nodeId) or get_screenshot(fileKey) — Claude gets real Figma data back as structured JSON without you having to export anything manually.

En el framework WAT, los servidores MCP son Tools (Herramientas) — pero más ricos que los scripts de Python simples. En lugar de llamar a python tools/mi_script.py, Claude llama directamente a un endpoint MCP.

El servidor Figma MCP, por ejemplo, le permite a Claude llamar a get_design_context(nodeId) o get_screenshot(fileKey) — Claude recibe datos reales de Figma como JSON estructurado sin que vos tengas que exportar nada manualmente.

Key Insight

MCP is what makes Claude Code actually agentic. It's the difference between Claude knowing about your system and Claude being able to act on your system.

Idea Clave

MCP es lo que hace que Claude Code sea verdaderamente agéntico. Es la diferencia entre que Claude sepa sobre tu sistema y que Claude pueda actuar sobre tu sistema.

    Preguntale a Claude Code: "¿Qué servidores MCP están configurados en este proyecto? Mostrame la configuración." — fijate en los archivos .claude o de configuración que Claude muestre.
  2. Read the Figma MCP server documentation — ask Claude: "Explain what the Figma MCP server can do, using the WAT framework as context. What would be the SQL equivalent of the get_design_context call?"
