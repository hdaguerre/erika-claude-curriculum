Week 3 Day 14

Skills — Structure, Triggers & the Stored Procedure Analogy

Estimated time: ~2.5 hours
ConceptsConceptos

1. What a skill is1. Qué es una habilidad

A skill is a self-contained unit of instructions that Claude loads when triggered. It lives in its own folder inside skills/. At the center of every skill is a SKILL.md file with two parts:

  • YAML front matter (at the top, between --- markers) — a short header with name and description
  • Body — the step-by-step instructions Claude follows when the skill is active

Skills can also have supporting folders: references/ for knowledge documents, scripts/ for executable code, and assets/ for templates and files.

Una habilidad (skill) es una unidad autónoma de instrucciones que Claude carga cuando se activa. Se almacena en una carpeta con un archivo SKILL.md en su núcleo. El SKILL.md tiene un encabezado YAML (nombre y descripción) y un cuerpo (instrucciones sobre qué hacer). Cuando la descripción coincide con algo que Erika pide, Claude carga y sigue la habilidad automáticamente.

SQL Connection

A skill is a stored procedure. The YAML header is the procedure signature — name and input parameters. The description field is the trigger condition (like a trigger's WHEN clause). The SKILL.md body is the procedure body — the actual logic that executes.

Conexión SQL

Una habilidad es un procedimiento almacenado. El encabezado YAML = la firma del procedimiento (nombre + parámetros). La descripción = la condición de activación (como la cláusula WHEN de un trigger). El cuerpo del SKILL.md = el cuerpo del procedimiento.

2. How skills trigger2. Cómo se activan las habilidades

Claude reads the description field of every skill in your skills/ folder. When you ask for something, Claude pattern-matches your request against all the descriptions and loads the matching skill automatically.

A weak description: "Use for captions."

A strong description: "Use this skill when writing Instagram captions for bakery/food content in Spanish, especially when the user wants a warm, community-focused tone for Mi Taller."

The difference is specificity. A well-written description tells Claude exactly when to trigger and when not to. The more specific the trigger, the more reliably Claude uses the skill.

El campo de descripción en el encabezado YAML es el disparador. Le dice a Claude cuándo usar esta habilidad. Una descripción bien escrita es específica: "Usar esta habilidad cuando el usuario pide escribir captions de Instagram para contenido de comida/repostería en español." Claude compara sus solicitudes con todas las descripciones de habilidades y carga la correcta.

Key Insight

Skills let Claude be an expert in something specific without you re-explaining it every session. Write a skill for "bakery captions in Spanish" once, and Claude will always follow those exact instructions whenever that context arises — in any project that has the skill.

Idea Clave

Las habilidades permiten que Claude sea experto en algo específico sin tener que re-explicarlo cada vez. Podría crear una habilidad para "escribir captions de repostería en español" y cada vez que esté en ese proyecto y necesite un caption, Claude automáticamente sigue las instrucciones de esa habilidad.

3. The skill-creator skill3. La habilidad skill-creator

This project has a special skill called skill-creator — it's a skill that builds new skills. When you invoke it with /skill-creator, it walks through a structured process: draft the skill, write test cases (evals), optimize the description for reliable triggering, and output a ready-to-use skill folder.

It's a meta-tool — a code generator that scaffolds stored procedures for you. You'll use it in Day 19.

Este proyecto tiene una habilidad especial llamada skill-creator que ayuda a construir NUEVAS habilidades. Es la habilidad que crea más habilidades — una meta-herramienta.

SQL Connection

The skill-creator is like a code generator that scaffolds new stored procedures for you — the same way SQL Developer's procedure wizard generates the boilerplate structure of a new stored procedure.

Conexión SQL

El skill-creator = un generador de código que crea scaffolding de nuevos procedimientos para vos. Igual que un asistente de código en SQL Developer que genera la estructura de un procedimiento almacenado.

Exercise (~45 minutes)Ejercicio (~45 min)
  1. Open VS Code with the erika project. In the Explorer, navigate to skills/skill-creator/. Open SKILL.md. Read the YAML front matter carefully — notice the name and description fields. What does the description tell you about when this skill triggers?
    Abrí VS Code con el proyecto erika. En el Explorador, navegá hasta skills/skill-creator/. Abrí SKILL.md. Leé el encabezado YAML con atención — fijate en los campos name y description. ¿Qué te dice la descripción sobre cuándo se activa esta habilidad?
  2. Read the first 50 lines of the SKILL.md body. Notice the structure: how does it organize its instructions? Does it reference any files in a references/ subfolder? How does it differ from a plain CLAUDE.md?
    Leé las primeras 50 líneas del cuerpo del SKILL.md. Fijate en la estructura: ¿cómo organiza sus instrucciones? ¿Hace referencia a algún archivo en una subcarpeta references/? ¿En qué se diferencia de un CLAUDE.md simple?
  3. Open skills/skill-creator/evals.json if it exists. Read 2-3 examples. Evals are test cases — they describe a user request and the expected skill behavior. This is how skills are tested and improved over time. Does anything surprise you?
    Abrí skills/skill-creator/evals.json si existe. Leé 2-3 ejemplos. Los evals son casos de prueba — describen una solicitud del usuario y el comportamiento esperado de la habilidad. Así es como se prueban y mejoran las habilidades con el tiempo. ¿Algo te sorprende?
  4. Now, without writing any code, sketch your own "bakery caption" skill on paper or in a plain text file. Write just the YAML header:
    • What would the name be? (kebab-case, e.g., bakery-caption)
    • What would the description say? Be specific — when exactly should this skill trigger? What contexts? What NOT to trigger for?
    This is design thinking before implementation. Don't worry about the body yet.
    Ahora, sin escribir código, bosquejá tu propia habilidad de "caption de repostería" en papel o en un archivo de texto. Escribí solo el encabezado YAML:
    • ¿Cuál sería el name? (kebab-case, por ejemplo, bakery-caption)
    • ¿Qué diría la description? Sé específica — ¿cuándo exactamente debería activarse esta habilidad? ¿En qué contextos? ¿Para qué NO debería activarse?
    Esto es pensar el diseño antes de implementarlo. No te preocupes por el cuerpo todavía.
  5. Optional: Open the terminal, navigate to the erika project root, and type claude. Ask it: "What skills do you have available?" See how it responds.
    Opcional: Abrí la terminal, navegá hasta la raíz del proyecto erika y escribí claude. Preguntale: "What skills do you have available?" Observá cómo responde.
