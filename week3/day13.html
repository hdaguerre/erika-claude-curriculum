CLAUDE.md — What It Is & What Gets Loaded

Estimated time: ~2.5 hours
ConceptsConceptos

1. What CLAUDE.md is1. Qué es CLAUDE.md

CLAUDE.md is a special file that Claude Code reads automatically every time you start a session in that project folder. It's the first thing Claude reads — before any other files. It sets the entire context for the session: who Claude is in this project, what rules it follows, what the project is about, and how to operate.

Think of it as the briefing document Claude gets before starting work. If you want Claude Code to know something about your project — the tone to use, the constraints to respect, the key files to be aware of — you write it in CLAUDE.md.

CLAUDE.md es un archivo especial que Claude Code lee automáticamente cuando abrís una carpeta de proyecto. Es lo primero que Claude lee — antes que cualquier otro archivo. Establece el contexto para toda la sesión: quién es Claude en este proyecto, qué reglas sigue, de qué trata el proyecto y cómo operar dentro de él.

SQL Connection

CLAUDE.md is the login.sql — the file that Oracle runs automatically when you start a SQL*Plus session. In Oracle, login.sql sets session parameters, NLS settings, and context. CLAUDE.md does the same: it runs first, every time, no exceptions. It's also analogous to a DDL file that defines the schema — it tells the system what everything is before any work begins.

Conexión SQL

CLAUDE.md = el archivo DDL que corre al inicio de cada sesión de base de datos. En Oracle, podrías tener un login.sql que establece parámetros de sesión y contexto. CLAUDE.md es ese archivo — corre primero, siempre, sin excepciones.

Key Insight

CLAUDE.md is the most powerful file in any Claude Code project. If Claude Code isn't behaving the way you want, the answer is almost always: update CLAUDE.md. It's your first lever for shaping Claude's behavior.

Idea Clave

Si quiere que Claude Code sepa algo sobre su proyecto, lo escribe en CLAUDE.md. Es el archivo más poderoso de cualquier proyecto.

2. What actually gets loaded into context2. Qué se carga realmente en el contexto

When you type claude in a project folder, Claude Code reads:

  • CLAUDE.md — always, automatically, first
  • Files you reference in conversation — when you say "look at this file," Claude reads it
  • Files Claude needs to act on — if you ask Claude to edit a file, it reads it

Claude does not automatically read every file in your project. Your codebase could be gigabytes — that would be impossible. CLAUDE.md is the guaranteed-read file. Everything else is on demand.

Cuando Erika escribe claude en una carpeta de proyecto, Claude Code lee: CLAUDE.md primero, y luego los archivos que ella mencione en la conversación. NO lee TODOS los archivos automáticamente — solo lo que ella le indique. CLAUDE.md es el archivo de lectura garantizada.

3. What a good CLAUDE.md contains3. Qué contiene un buen CLAUDE.md

A well-written CLAUDE.md typically covers:

  • Project purpose — what this project is and why it exists
  • How to operate — key rules, workflows, what tools to use
  • Key files to know about — the important files Claude should be aware of
  • Constraints — things Claude should never do, tone to use, formats to follow

It's written in plain language — no special syntax needed. Write it like you're briefing a smart new colleague on the first day.

Un buen CLAUDE.md tiene: propósito del proyecto (qué es esto), cómo operar (reglas clave), archivos clave que conocer, cualquier restricción. Se escribe en lenguaje simple — no se necesita sintaxis especial.

Exercise (~60 minutes)Ejercicio (~60 min)
  1. Open VS Code. In the Explorer panel, find CLAUDE.md at the root of the erika project. Click it to open. Read it fully — don't skim. Notice: what sections does it cover? What tone does it use? What does it tell Claude about how to operate?
    Abrí VS Code. En el panel Explorador, encontrá el CLAUDE.md en la raíz del proyecto erika. Hacé clic para abrirlo. Leélo completo — no lo ojees. Fijate: ¿qué secciones cubre? ¿Qué tono usa? ¿Qué le dice a Claude sobre cómo operar?
  2. Note the structure as you read. How many sections? What's in the YAML-style front matter (if any)? What rules does it give Claude? What's the most important thing it communicates?
    Tomá nota de la estructura mientras leés. ¿Cuántas secciones tiene? ¿Qué hay en el encabezado estilo YAML (si lo hay)? ¿Qué reglas le da a Claude? ¿Cuál es lo más importante que comunica?
  3. Now create a new folder on your Desktop (or wherever makes sense): call it mi-taller. This will be the home for your personal creative project — your bakery/Instagram content.
    Ahora creá una nueva carpeta en tu Escritorio (o donde tenga sentido): llamala mi-taller. Este va a ser el hogar de tu proyecto creativo personal — tu panadería/contenido de Instagram.
  4. In VS Code, File → Open Folder → open mi-taller. Create a new file: click the new-file icon in the Explorer, name it CLAUDE.md.
    En VS Code, Archivo → Abrir Carpeta → abrí mi-taller. Creá un nuevo archivo: hacé clic en el ícono de nuevo archivo en el Explorador, nombralo CLAUDE.md.
  5. Write your own CLAUDE.md from scratch. It should include:
    • What "Mi Taller" is — your personal creative project (bakery, handmade goods, Instagram content — describe it in your own words)
    • Your audience — Spanish-speaking followers, the vibe you're going for
    • Content rules — tone, style, what to avoid (salesy language? formal Spanish? certain topics?)
    • Key reference files you'll add later (brand voice doc, recipe list, caption library)
    • How Claude should behave when working in this project
    Escribí tu propio CLAUDE.md desde cero. Debería incluir:
    • Qué es "Mi Taller" — tu proyecto creativo personal (panadería, artesanías, contenido de Instagram — describilo con tus propias palabras)
    • Tu audiencia — seguidores de habla hispana, el vibe que buscás
    • Reglas de contenido — tono, estilo, qué evitar (¿lenguaje de ventas? ¿español formal? ¿ciertos temas?)
    • Archivos de referencia clave que vas a agregar después (documento de voz de marca, lista de recetas, biblioteca de captions)
    • Cómo debe comportarse Claude cuando trabaja en este proyecto
  6. Save the file. Open the terminal, navigate into mi-taller, and type claude. Claude Code will read your CLAUDE.md automatically. Say hello and ask Claude to introduce itself based on what it read. Does it sound like it understood your project?
    Guardá el archivo. Abrí la terminal, navegá hasta mi-taller y escribí claude. Claude Code va a leer tu CLAUDE.md automáticamente. Saludá y pedile a Claude que se presente en base a lo que leyó. ¿Suena como si hubiera entendido tu proyecto?
