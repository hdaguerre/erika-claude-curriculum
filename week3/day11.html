Week 3 Day 11

Claude Code vs Claude Desktop

Estimated time: ~2 hours
ConceptsConceptos

1. What Claude Code actually is1. ¿Qué es Claude Code realmente?

Claude Code is a command-line interface (CLI) — a way to interact with Claude directly from your terminal or VS Code, where Claude can read and write files on your actual computer, run scripts, and work inside a real project folder.

Unlike Claude Desktop (where you chat and Claude helps you think), Claude Code is where Claude does things. It can create files, edit code, run commands, and navigate your whole project. It's the difference between asking someone for advice and actually handing them the keyboard.

Claude Code es una interfaz de línea de comandos (CLI) — una forma de interactuar con Claude directamente desde tu terminal o VS Code, donde Claude puede leer y escribir archivos en tu computadora, ejecutar scripts y trabajar dentro de una carpeta de proyecto real.

A diferencia de Claude Desktop (donde chateas y Claude te ayuda a pensar), Claude Code es donde Claude hace cosas. Puede crear archivos, editar código, ejecutar comandos y navegar todo tu proyecto. Es la diferencia entre pedirle consejo a alguien y directamente darle el teclado.

SQL Connection

Claude Code is like SQL*Plus or a command-line database client — instead of clicking through a GUI, you type commands and the engine executes them directly. Claude Desktop is like using Oracle Application Express (APEX) — a friendly interface. Claude Code is the raw terminal.

Conexión SQL

Claude Code es como SQL*Plus o un cliente de base de datos en línea de comandos — en vez de hacer clic en una interfaz gráfica, escribís comandos y el motor los ejecuta directamente. Claude Desktop es como usar Oracle Application Express (APEX) — una interfaz amigable. Claude Code es la terminal en crudo.

Key Insight

The GUI (Claude Desktop) is for thinking together. The CLI (Claude Code) is for building together. You need both.

Idea Clave

La interfaz gráfica (Claude Desktop) es para pensar juntos. La línea de comandos (Claude Code) es para construir juntos. Necesitás las dos.

2. VS Code — your new workspace2. VS Code — tu nuevo espacio de trabajo

VS Code is a code editor — think of it like SQL Developer or Toad, but built for all programming languages, not just SQL. It has three main panels you'll use every day:

  • Explorer (left sidebar) — shows all files and folders in your project, exactly like the folder tree in SQL Developer.
  • Editor (center) — where you view and edit files. Multiple files can be open as tabs.
  • Terminal (bottom) — a command line built right into the editor. This is where you'll run Claude Code. Open it with Ctrl+` (backtick) or View → Terminal.

VS Code es un editor de código — pensalo como SQL Developer o Toad, pero para todos los lenguajes de programación, no solo SQL. Tiene tres paneles principales que vas a usar todos los días:

  • Explorador (barra lateral izquierda) — muestra todos los archivos y carpetas de tu proyecto, exactamente como el árbol de carpetas en SQL Developer.
  • Editor (centro) — donde ves y editás archivos. Pueden estar abiertos varios archivos como pestañas.
  • Terminal (abajo) — una línea de comandos integrada en el editor. Aquí es donde ejecutarás Claude Code. Abrilo con Ctrl+` (acento grave) o Ver → Terminal.
SQL Connection

VS Code's terminal is like the SQL*Plus command prompt — text in, text out, direct execution. The Explorer sidebar is like the object tree in your SQL client.

Conexión SQL

La terminal de VS Code es como el prompt de SQL*Plus — texto de entrada, texto de salida, ejecución directa. La barra lateral del Explorador es como el árbol de objetos en tu cliente SQL.

3. Today's goal: just look around3. Tu objetivo de hoy: solo explorar

Today you are not building anything. You are becoming familiar with the project's geography. Open VS Code, open the erika project folder, and click through every file. Read the CLAUDE.md. Read the README if there is one. Don't change anything — just understand what's here.

This is exactly what a senior developer does when they join a new project: they read the code before they write any.

Hoy no vas a construir nada. Vas a familiarizarte con la geografía del proyecto. Abrí VS Code, abrí la carpeta del proyecto erika y hacé clic en cada archivo. Leé el CLAUDE.md. Leé el README si hay uno. No cambies nada — solo entendé qué hay aquí.

Esto es exactamente lo que hace un desarrollador senior cuando llega a un proyecto nuevo: lee el código antes de escribir una sola línea.

Exercise (~45 minutes)Ejercicio (~45 minutos)
  1. Open VS Code. If you don't have it installed, download it free from code.visualstudio.com. Once open, go to File → Open Folder.
    Abrí VS Code. Si no lo tenés instalado, descargalo gratis desde code.visualstudio.com. Una vez abierto, andá a Archivo → Abrir Carpeta.
  2. Navigate to your erika project folder: C:\Users\[you]\Desktop\claude-projects\erika (or wherever it lives on your machine). Click "Select Folder."
    Navegá hasta tu carpeta del proyecto erika: C:\Users\[vos]\Desktop\claude-projects\erika (o donde sea que esté en tu máquina). Hacé clic en "Seleccionar Carpeta."
  3. In the Explorer panel on the left, click through every file in the project. Read at least 3 files fully — especially CLAUDE.md. Notice its structure: what it covers, how it's written, what tone it uses.
    En el panel Explorador a la izquierda, hacé clic en cada archivo del proyecto. Leé al menos 3 archivos completos — especialmente el CLAUDE.md. Fijate en su estructura: qué cubre, cómo está escrito, qué tono usa.
  4. Open the integrated terminal: press Ctrl+` (the backtick key, top-left of keyboard) or go to View → Terminal. A panel opens at the bottom.
    Abrí la terminal integrada: presioná Ctrl+` (la tecla de acento grave, arriba a la izquierda del teclado) o andá a Ver → Terminal. Se abre un panel abajo.
  5. Type this command to list files in the current folder — on Windows use: dir. On Mac/Linux use: ls. Press Enter. You'll see the file list appear.
    Escribí este comando para ver los archivos en la carpeta actual — en Windows usá: dir. En Mac/Linux usá: ls. Presioná Enter. Vas a ver la lista de archivos aparecer.
  6. Navigate into a subfolder: type cd curriculum and press Enter. Then type dir (or ls) again to see what's inside.
    Navegá a una subcarpeta: escribí cd curriculum y presioná Enter. Después escribí dir (o ls) de nuevo para ver qué hay adentro.
  7. Go back up: type cd .. and press Enter. You're back at the root of the project. Type dir one more time to confirm.
    Volvé atrás: escribí cd .. y presioná Enter. Estás de vuelta en la raíz del proyecto. Escribí dir una vez más para confirmar.
